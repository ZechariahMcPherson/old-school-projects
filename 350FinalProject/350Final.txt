	The challenged posed to us in this assignment is to be able to search through a query of data, using a given similarity function, recognize and discern AU1, AU12, and AU17. These points of data represent raising inner brow, raising lip corners, and a raised chin, respectively. The goal of this code is to be able to recognize facial expression and be able to compare the similarities to its surrounding files. To accomplish this, we were instructed to apply a given similarity function as well as our knowledge gained through this course. This folder, appropriately is named DATA and without this key information, the code will not run properly.
	The methods used in this solution first and foremost assume that all data is contained within the same folder. Then we use a for-loop to loop through the first 10 values to convert into a string. Because of how 10 is viewed. It was easy to loop through 10 templates and 30 queries, 1 through 9, to match up with the file name which is written 001,002,etc.  For the next 37 templates and 111 queries, the for-loop iterates through and assigns the i value to 0i+rest of file name to properly identify each file. The Query files are separated by AU1,AU12 and AU17, separated into Q1,Q2,Q3 respectively. Now each is a string and can be further applied in the code. 
The next step is utilizing a templateFile to give the size of all the template data that can then be referenced to allocate memory to RAM. Then the buffer is used to populate a stringstream TemplateContents which populates a two dimensional array of all the templates. The buffer is subsequently deleted. Before the Solve function ends, we call the ReadQueryandDoSimilarity() function for each of the query lists. Inside said function, a similar process is used as the templateFile except now it is a QueryFile, following the same format as the previous templateFile read in. The queries are filed into FileInContents using the same allocated memory from earlier as populates a Query array. Following the given algorithm, we begin calculating similarity. A for loop is created and the value at I and j for the templateArray is assigned to TempVal, as a way to multiply against the value of the Query array. This is done to use only one square root function which will be explained later. After all is calculated, the value found is pushed into a vector defined in our .h file. The numbers variables are cleared and the process loops until all data is in the vector. With all the values, now all that is left is to sort them. Using a standard sort function and a Boolean compare that compares by the second value (not the template file), all results are sorted. All that is left, is a print function which writes using standard ios writes an out file and formats the data for readability. Before closing the function, the vector and the buffer is once again deleted. Leading to a much cleaner running program.

In order to proceed with optimal results, we implemented various different methods. These methods include allocating memory space in RAM. By taking the file’s size and using it as the reference of how much memory to allocate. Then when the memory is no longer required, then that memory is deleted and the buffer is removed. This avoids using hard drive space and taking more time. RAM is simply faster. The clearing of the buffer is key in order to avoid unnecessary variables and reallocation.  Another way to expedite the code was by using one square root function in the similarity function instead of using two. The less operations the algorithm performs, the less time it will take to complete. Because of how the square root function is utilized, implementing it over the product of both summations will still yield the same result. The sort function is from standard libraries. If we had written our own, it very may well have been a more inefficient algorithm.
